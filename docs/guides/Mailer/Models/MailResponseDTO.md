# MailResponseDTO

The model returned by `NotificationMessages` factory methods. It's pre-filled with the recipient, subject, and generated HTML body — ready to be mapped straight into a `MailRequestDTO`.

You typically won't construct this yourself. The `NotificationMessages` class builds it for you.

## Properties

| Property       | Type     | Notes                                                                                         |
| -------------- | -------- | --------------------------------------------------------------------------------------------- |
| `Receiver`     | `string` | The recipient's email address, passed through from your input                                 |
| `ReceiverName` | `string` | Recipient's name, passed through from your input                                              |
| `Subject`      | `string` | Pre-set subject line for the specific template (e.g. `"Email Verification"`, `"Login Alert"`) |
| `Body`         | `string` | The fully rendered HTML email body generated by the template                                  |

## Typical Usage

```csharp
var emailContent = NotificationMessages.RegistrationConfirmationMailNotification(
    receiver: user.Email,
    name: user.FirstName,
    token: verificationToken,
    senderName: "YourApp"
);

// Map the MailResponseDTO into your send request:
var request = new MailRequestDTO
{
    Sender = "noreply@yourdomain.com",
    SenderName = "YourApp",
    Receiver = emailContent.Receiver,      // from MailResponseDTO
    ReceiverName = emailContent.ReceiverName,
    Subject = emailContent.Subject,
    Body = emailContent.Body
};
```
